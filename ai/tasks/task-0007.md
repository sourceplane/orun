## Objective
Open the `internal/revision` package and ship the **typed model, the revision-key
generator with collision resolution, and the writer skeleton** that executes the
ordered writes from `design.md` §5.1 / `cli-surface.md` §1.2:

1. write `trigger.json`
2. write `revision.json`
3. write `plan.json`
4. update `refs/latest-revision.json` via CAS (caller-owns-retry per `state-store.md` §6)
5. update `refs/triggers/<triggerName>/{latest.json, <scope>.json}` (typed `TriggerRef`)
6. write `indexes/revisions/<revisionKey>.json` via `CreateIfAbsent`
7. on first new-layout creation, write `.orun/version.json` (`StateStoreVersion`, layout `"revision-first"`, version `1`)

Manifest writes and `ResolveRevision` (the seven-branch resolver) are
**deferred to PR-B (Task 0008)** — keep this PR small and focused.

The writer must be gated by an internal `stateCompatibilityWrites bool` flag
(default `true`), threaded via the writer config struct. In this PR the
compatibility branch is a clearly-marked **stub** that returns `nil` without
writing legacy `.orun/plans/<checksum>.json` — actual legacy mirroring is wired
during M5 CLI rewire. The flag plumbing must exist now so M5 has nothing to
add but the legacy-write body.

## Required Outcomes
- `internal/revision` package compiles, vet-clean, race-clean.
- `WriteRevision` is the **only** exported writer entry point. It executes the seven-step order exactly as listed above, in that order. Each step's failure path returns the wrapped sentinel from `internal/statestore` — no new error sentinels.
- CAS on `refs/latest-revision.json` follows the caller-owns-retry pattern from `state-store.md` §6: read once, attempt CAS, retry up to a small bounded number (e.g. 5) on `ErrConflict` re-reading meta each time; surface the final `ErrConflict` to the caller if all retries lose.
- Revision-key collision is resolved via `CreateIfAbsent` on `indexes/revisions/<key>.json`: first writer wins; loser falls back to `<key>-x1`, `<key>-x2`, … capped at e.g. `-x99` after which return `ErrConflict`-wrapped failure. Two concurrent `WriteRevision` calls with the same `(trig, planHash)` MUST produce two distinct revision keys.
- `EnsureStateStoreVersion` is idempotent: first call writes; subsequent calls observe `ErrExists` and return `nil`. Concurrent first-callers MUST converge to a single `version.json` (verified by test).
- Deterministic JSON for every persisted artifact: `Encoder.SetIndent("", "  ")`, `SetEscapeHTML(false)`, trailing `\n`. Cover with at least one `internal/testfx/statefs.AssertJSONFile` test per persisted file shape (`trigger.json`, `revision.json`, `plan.json`, `indexes/revisions/<key>.json`, `version.json`).
- All exported symbols carry doc comments.
- Package stays leaf-clean against everything except `internal/statestore`, `internal/triggerctx`, stdlib, and `github.com/oklog/ulid/v2`. `go list -deps ./internal/revision | grep "/orun/internal/"` returns only `internal/revision`, `internal/statestore`, `internal/triggerctx`.
