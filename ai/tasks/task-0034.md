## Objective

Implement the read side of `internal/catalogstore` per
`specs/orun-component-catalog/catalog-store.md` §4 (reader fallback) and §8
(index rebuild). Wire all five Resolver methods on the `*store` receiver with
the documented fallback chain, and add `RebuildIndexes` so that
`indexes/components/*.json`, `indexes/catalogs/*.json`, and
`indexes/sources/*.json` can be rebuilt byte-identically from the source-tree
truth. This closes Milestone C4.

## Required Outcomes

- [ ] `ResolveCurrentSource(ctx)` reads `refs/sources/current.json`. On
      `errors.Is(err, statestore.ErrNotFound)`, falls back to walking
      `sources/*/source.json` and returns the entry with the most recent
      `createdAt`. Returns `ErrCatalogNotFound` if the walk yields zero
      entries.
- [ ] `ResolveSource(ctx, sel)` honours all five `RefSelector.Kind` values
      (`current`, `main`, `latest`, `branch`, `pr`) and reads the
      corresponding `refs/sources/...json` via the existing
      `paths.SourceRefPath` / `SourceBranchRefPath` / `SourcePRRefPath`
      helpers. Reads the `SourceSnapshot` body at `SourceDocPath`.
      Returns wrapped `ErrCatalogNotFound` when the ref is missing AND no
      walk-fallback matches. `Snapshot` field is rejected with a clear
      "not yet implemented (C8)" error envelope.
- [ ] `ResolveCatalog(ctx, sel)` reads the relevant `refs/catalogs/...json`,
      then loads the `CatalogSnapshot` body at `CatalogDocPath`. Falls back
      to "most recent `catalogs/*/catalog.json` under the resolved source"
      when the ref is missing. `current` resolves source first via
      `ResolveCurrentSource`. Returns `ErrCatalogNotFound` on full miss.
- [ ] `ResolveComponent(ctx, sel, name)` resolves the catalog, then reads
      `ComponentManifestPath(srcKey, catKey, name)`. Returns
      `ErrComponentNotFound` (NOT `ErrCatalogNotFound`) on missing manifest
      with the catalog itself present. Validates `name` via
      `catalogmodel.ValidateComponentName` (or the closest existing
      validator) before path construction.
- [ ] `ResolveComponentLatest(ctx, key)` reads
      `indexes/components/<sanitizedKey>.json` and returns the
      `ComponentLatest` view. **Fallback path:** on `ErrNotFound`, walk
      `sources/*/catalogs/*/components/<derivedName>/manifest.json`
      filtered by the source-scope rule from §4 to reconstruct an
      ad-hoc `ComponentLatest`. Document the walk's cost in a comment.
- [ ] `RebuildIndexes(ctx)`: walk all
      `sources/*/catalogs/*/{catalog.json,components/*/manifest.json}`,
      reconstruct `ComponentGlobalIndex`, `CatalogGlobalIndex` (if such
      a body type exists; otherwise a documented placeholder per §8),
      `SourceGlobalIndex`, then write each via `StateStore.Write` (plain
      overwrite — not CAS). Output MUST be byte-identical to what the
      Writer surface produces for the same tree (T-STORE-3); use the same
      `catalogmodel.PrettyEncode` encoder and the same merge ordering as
      `mergeComponentGlobalIndex` from `indexes.go`.
- [ ] **Reader fallback ladder per §4 is preserved exactly** — refs first,
      walk second, sentinel error on miss. No reader EVER reads from
      `.orun/revisions/<revKey>/` (Phase 1 layout) without first attempting
      the catalog-tree path; this is the C4-locked invariant for Phase 2.
- [ ] All path construction goes through `paths.go` helpers — no raw
      `filepath.Join` / string concatenation. `grep -RE '^\s*"(os|io/ioutil|path/filepath)"' internal/catalogstore/`
      stays empty.
- [ ] `errors.Is` continues to chain through `statestore.ErrNotFound`,
      `statestore.ErrInvalid`, etc., preserving the wrapping pattern PR-1
      and PR-2 established (`fmt.Errorf("%w: %w", ErrCatalogNotFound,
      statestore.ErrNotFound)` style).
- [ ] **Coverage target:** `internal/catalogstore` ≥ 91 % (floor 90 %;

## Constraints

1. Only `internal/catalogstore/` is touched. Touching any other
   `internal/*` or `cmd/orun/` package is forbidden in this PR.
2. `internal/catalogstore` MUST NOT import `os`, `io/ioutil`, or
   `path/filepath`. All FS access flows through `statestore.StateStore`.
3. Body decoding MUST use `catalogmodel.PrettyEncode` /
   `catalogmodel.PrettyDecode` (whichever is paired) — bodies are
   round-trippable through this canonical encoder for byte-identical
   rebuild.
4. The `Resolver.RebuildIndexes` addition is the ONLY interface change
   permitted. Do NOT widen any other interface or method signature.
5. Retry / CAS budgets remain consistent with PR-2 (16). `RebuildIndexes`
   uses plain `Write` per spec §8 — no CAS.
6. Test fixtures live under `internal/catalogstore/testdata/` if needed;
   prefer in-line fixture builders to keep the diff narrow and reviewable.
7. Walk operations honour `ctx` cancellation; tests inject a cancelled
   context and assert prompt return.
8. Forbidden imports/packages: `internal/runtime`, `cmd/orun`,
   `internal/catalogsync`, anything network-related.
