## Objective

Land Milestone M2 PR A from
`specs/orun-state-redesign/implementation-plan.md` §M2: introduce
`internal/statestore` with the **frozen `StateStore` interface**, the error
taxonomy, the path-helper module, and a local-driver implementation of the
non-CAS operations (`Read`, `Write`, `CreateIfAbsent`, `Delete`). This is the
first production consumer surface for the trigger-first revision-first layout
and freezes the contract that M3 (`internal/revision`) and M4
(`internal/executionstate`) will build on.

You have latitude to also land PR B (`CompareAndSwap` + `List` + atomicity
property tests) and PR C (typed refs + indexes helpers) in this same PR if the
total diff stays reviewable (~≤ 700 production LOC excluding tests) and the

alone and report PR B / PR C as the next implementer task(s).

## PR Boundary

In scope:
- `internal/statestore/store.go` — `StateStore` interface, `ObjectMeta`,
  `ObjectInfo`, `WriteOptions` exactly as published in `state-store.md` §1.
  Doc comments on every exported symbol.
- `internal/statestore/errors.go` — `ErrNotFound`, `ErrExists`, `ErrConflict`,
  `ErrInvalid` sentinels (per `state-store.md` §4) plus internal wrap helpers
  used by the local driver. All driver errors MUST wrap a sentinel via
  `fmt.Errorf("%w: …", ErrX, …)` so callers can use `errors.Is` / `errors.As`.
  String sniffing is banned.
- `internal/statestore/paths.go` — every helper listed in `state-store.md`
  §2.1, including `EventPath`, all `*RefPath` variants, and both index helpers.
  Centralized validation of the allowed alphabet (`[a-zA-Z0-9._-]` per segment,
  no `..`, no leading `/`, no Windows separators). Raw string concatenation by
  callers is forbidden — `paths.go` is the only place that builds logical
  paths.
- `internal/statestore/local.go` — `LocalConfig`, `LocalStore`,
  `NewLocalStore(LocalConfig) (*LocalStore, error)`. Implements `Root`,
  `Read`, `Write` (atomic temp + fsync + rename with cross-device copy
  fallback), `CreateIfAbsent` (`O_EXCL`), `Delete` (no-op on absent, refuses
  non-empty directories). No global state.
- Orphan-tempfile cleanup at construction: best-effort sweep of
  `.orun-tmp-*` older than 1 hour. Failures during the sweep are logged at
  debug level (or recorded via a structured error return — your choice; just
  document it) and never prevent `NewLocalStore` from succeeding.
- `internal/statestore/local_test.go` — unit tests for the implemented
  operations including:
  - `Write` round-trip + `ObjectMeta.Revision == sha256(data)`.
  - `Write` is atomic: a concurrent reader either sees the previous bytes or
    the new bytes, never partial. Use `internal/testfx/statefs` for the
    workspace and assertions.
  - `CreateIfAbsent` returns `ErrExists` on second call; `errors.Is` succeeds.
  - `Read` of missing path returns `ErrNotFound`; `errors.Is` succeeds.
  - `Delete` of missing path is a no-op (no error).
  - `Delete` refuses non-empty directories (returns `ErrInvalid`).
  - Orphan tempfile older than 1 hour is removed on construction; tempfile
    younger than 1 hour is preserved.
  - Path validation: `..`, leading `/`, empty segment, invalid character all
    return `ErrInvalid` from `Write` / `Read` / `CreateIfAbsent` / `Delete`.
- `internal/statestore/paths_test.go` — table-driven tests for every helper
  in §2.1, plus the validation alphabet.
- Update `Makefile` `test-state-redesign` target to include
  `./internal/statestore/...` with `-cover` and a coverage threshold check
  (≥ 95% statement coverage for this package as `test-plan.md` §1 requires —
  if the target currently only echoes the package list, expand it to invoke
  `go test -cover` and parse the result; an existing helper is fine).
- `go.mod` / `go.sum`: run `go mod tidy` after the change. No new direct
  dependencies are expected for PR A — the existing `oklog/ulid/v2`,
  `pgregory.net/rapid` set is sufficient.

Out of scope (must be explicit non-goals — see below).

- **No production callers wired.** Do not touch `cmd/orun/`, `internal/state`,
  `internal/runner`, or any other package. This PR introduces a layer; M3 and
  M4 will consume it.
- **No CLI surface changes.** No new flags, no new commands.
- **No legacy state writes or migrations.** Compatibility writes are an M3+
  concern (`compatibility-and-migration.md`).
- **No `clock` injection refactor.** If `LocalStore` needs a clock for the
  orphan-sweep age threshold, accept an optional `Clock func() time.Time`
  field on `LocalConfig` defaulting to `time.Now`. Do not introduce a repo-wide
  `clock.Clock` interface in PR A — that is a cross-cutting Milestone M0
  follow-up the spec calls out separately.
- **No JSON schema enforcement.** `StateStore` is bytes-in / bytes-out. The
  refs/manifest/index JSON shapes are M3's responsibility.
- **No event emission.** `execution-created` and `bridge-mirror-failed` events
  land with M4.

## Constraints

- The `StateStore` interface signatures from `state-store.md` §1 are frozen by
  the end of this PR. Any divergence requires a `/ai/proposals/task-0003-spec-update.md`
  proposal and the spec update before merge.
- Path policy is centralized: callers MUST go through `paths.go`. The local
  driver itself uses the same helpers internally where applicable.
- All driver errors wrap a sentinel; `errors.Is` / `errors.As` are the only
  supported caller checks.
- Atomic `Write`: temp file in the same directory as the target; `fsync` the
  tempfile before `os.Rename`; on `EXDEV` (cross-device) fall back to
  copy + fsync + rename within a tempdir on the target FS, then atomic rename.
- No `panic()` in production paths.
- No `time.Now()` directly in production code outside of the `LocalConfig.Clock`
  default — wire everything through the configured clock so tests can drive
  the orphan-sweep boundary deterministically.
- No log lines containing absolute filesystem paths beyond what is already
  surfaced by `LocalStore.Root()` for diagnostics, and never include user
  bytes (object contents) in logs.
- New JSON output not introduced in PR A (deferred to M3); when you add it
  later, stable key order + trailing newline.
- `internal/statestore` must not import any other `internal/` package. Like
  `internal/testfx/statefs`, it sits at the leaf of the new state layer.
- Test helpers MUST use `internal/testfx/statefs.NewWorkspace`; do not
  hand-roll temp dirs.

## Integration Notes

- M3 (`internal/revision`) will be the first consumer. The interface you ship
  here is the interface M3 codes against; the contract freeze is real.
- M4 (`internal/executionstate`) will add a `Bridge` that uses
  `LocalStore.Write` to mirror legacy runner output. PR A does not need to
  account for the bridge beyond ensuring `Write` is safe under concurrent
  use.
- Remote drivers (Phase 2, R2/S3) will reuse the same interface. Keep the
  local driver's tempfile + rename pattern out of the interface contract —
  it's an implementation detail.
