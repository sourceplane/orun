Release notes for orun v2.34.0 (also covering v2.33.0, which shipped without
notes).

## Highlights

- **The next-generation cockpit, in preview: `orun tui-next`.** A ground-up
  rebuild of the TUI as the terminal head of orun cloud
  (`specs/orun-tui-v2`). Both heads consume the same state store and fold the
  same event vocabulary, so what you see in the terminal is what the console
  shows — at terminal density. The v1 cockpit remains the default
  (`orun` / `orun tui`) while v2 soaks; opt in with `orun tui-next`,
  `orun tui --next`, `orun agent --next`, or `ORUN_TUI=next`.

  - **Six product surfaces matching cloud nav** — Home (stat tiles, the
    needs-attention fold, latest activity), Work (approval-sealed epics with
    the milestone ladder and session links), Agents (the interactive
    conversation head: streaming turns, tool cards, sticky approval cards,
    always-on composer), Activity (runs feed → run → job → step log, with
    facets and an errors-only log filter), Catalog (change overlay,
    component detail, and the Compose flow that replaces Plan Studio), and
    Events — plus a design-system Gallery.
  - **Calm by construction.** An idle cockpit renders zero frames; motion
    only means work. Measured: idle animation frames cost ~20µs and a full
    220×60 repaint ~60µs, versus 5–8ms per idle tick in the v1 cockpit.
  - **Streams over polls.** Filesystem watches on the `.orun` ref
    directories and new step-level runner events replace the v1 cockpit's
    3s workspace reload and 600ms working-tree poll. Runs dispatched from a
    CLI in another terminal stream into the TUI live.
  - **One command grammar.** `:` (or `ctrl+k`) opens a command palette that
    contains every action; `?` help is generated from the same registry;
    `esc` always goes back and never quits.
  - **Cloud lane groundwork.** With `ORUN_BACKEND_URL` + `ORUN_WORKSPACE`
    and a linked `orun login` credential, org runs merge into the Activity
    feed with a provenance badge. Connection is a status, never a mode:
    signed out or offline simply renders local-only.

- **`orun agent --next`.** The agent front door can open the cockpit v2
  Agents surface directly — the sessions list and conversation head — while
  bare `orun agent` keeps the v1 surface until the flip.

- **Step-level run events.** The runner now exposes `OnStepStart` /
  `AfterStepTerminal` hooks, and the shared run service emits
  `step_started` / `step_completed` events — live views (and
  `status --watch` consumers) can render step progress from the stream
  instead of re-reading the working tree.

- **Secrets fix (v2.33.0):** run secrets resolve by wire ULID rather than
  the raw exec id, fixing remote-state resolution against the platform.

## Upgrade notes

- `.orun/` is now in `.gitignore`; four accidentally-tracked workspace-state
  files were removed from the repository.
- No breaking CLI changes. The v2 cockpit is opt-in; the v1 cockpit and all
  existing flags behave as before.
