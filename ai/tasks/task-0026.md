## Objective
Complete Milestone C2 by adding the post-load resolution stages — inference
(stage 6), dependency resolution (stage 8), validation (stage 9),
`manifestHash` computation (stage 10) — and a top-level `Resolve(ctx,
opts)` entry point that returns the full set of resolved (pre-graph)
`ComponentManifest`s with byte-stable per-component `manifestHash`.
Lands T-RES-1 (resolver determinism), T-RES-2 (provenance completeness),
and `ErrDependencyMissing` per `resolution-pipeline.md` §5–§8 and
`identity-and-keys.md` §10.

## Constraints
1. **PR-Boundary wording (from accepted spec proposal
   `ai/proposals/task-0025-spec-update.md`):** No edits to **existing
   source files in** `internal/catalogmodel/` or `internal/sourcectx/`.
   Additive sibling files (embed-only exports, small read-only typed
   views) needed by dependent packages are permitted; one additive
   file per cross-package contract surface, no logic.
2. All hashing inputs go through the canonical encoder in
   `internal/catalogmodel` — no `encoding/json` defaults for hashed
   payloads.
3. Inference is **additive** (per §4): explicit
   `runtime.inferred.*` lists in `component.yaml` are preserved and
   inference results are unioned in.
4. Heuristic framework detection uses a **fixed-order rule list**;
   tied matches resolve to the first match in the list (per §7).
5. The resolver itself MUST NOT call `time.Now()`; route any clock
   reads through a `clock.Time` seam (mirror the C1 `internal/sourcectx`
   adapter pattern).
6. `ResolvedCatalog` is a Phase-2-internal type local to
   `internal/catalogresolve` for this PR — do NOT pre-declare it in
   `internal/catalogmodel` (C3 will introduce `CatalogSnapshot`).
7. Determinism: `[]ComponentManifest` MUST be ordered by
   `componentKey`; all map-valued fields are encoded with sorted keys
   when hashed; file walks are deterministic.
8. `pgregory.net/rapid v1.1.0` (already in `go.mod`) for property
   tests where appropriate. Pin coverage with deterministic golden
   tests in addition to property tests, per the C0/C1 lesson
   (rapid-driven-only branches dropped catalogmodel coverage from
   91.1 % to 87.9 % on some seeds in Task 0024).
