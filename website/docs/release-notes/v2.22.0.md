Release notes for orun v2.22.0.

This release unifies every Orun surface on the **Orun Cloud brand** — an amber
accent (`#f59e0b`) over a neutral zinc canvas. The TUI cockpit, the CLI, the
documentation site, and the Orun Cloud console now render the same palette, so
"one design language, many surfaces" holds literally: the color you see in a CI
log is the color you see in the control room and on the docs page beside it.

## Highlights

- **Amber cockpit palette.** `internal/cockpit/style.DefaultPalette` — the single
  source of truth the TUI and every themed CLI surface read from — moves from the
  previous violet accent to the Orun Cloud amber (`#f59e0b` dark / `#d97706`
  light) over zinc neutrals. Status tokens (success, running, warning, error) are
  unchanged in meaning; only the brand accent and the neutral surfaces shifted.
- **Animated brand mark, re-toned.** The shimmer that sweeps the `◆ orun`
  wordmark now flows through an amber ramp instead of a violet one, staying inside
  the new brand family.
- **Docs match the product.** The documentation site
  ([orun-docs.pages.dev](https://orun-docs.pages.dev)) mirrors the same tokens, so
  the docs, the console, and the cockpit are visually one system.

## Nothing to change

This is a **presentation-only** release for the CLI/TUI: no flags, commands,
config fields, schemas, or plan outputs changed. `NO_COLOR` still strips color
while keeping glyphs, and terminals that flip light/dark pick up the matching
pair as before. Upgrade at your convenience.

## Design language

The palette lives in exactly one place. Reskinning the cockpit — or matching a
fork's brand — remains a single-file edit to `Palette`; see the
[cockpit overview](/cockpit/overview) and
[design principles](/principles) for how the token system spans the CLI, the
TUI, and the docs.
