### Key design choices

| Decision | Rationale |
|---|---|
| Shared monotonic entropy + `sync.Mutex` for ULID generation | `data-model.md ¬ß10` requires IDs to sort stably within the same millisecond across goroutines. |
| `TriggerKey` regex `^trg-[a-z0-9-]+-([a-f0-9]{7}\|local-dirty\|no-git)$` | Single source of truth shared by the generator and the rapid property test. |
| `GitSource` interface (not a concrete import) | Keeps `triggerctx` free of VCS dependencies; callers inject the git adapter. |
| Plan-scope defaults baked into system constructors | `manual‚Üífull`, `manual-changed‚Üíchanged`, `api‚Üífull`, `replay‚Üípassthrough`; dirty worktree overrides scope. Reduces caller boilerplate. |
| `ErrNoMatchingBinding` sentinel + `NoMatchingBindingError{Provider,Event,Action}` typed error wrapping it (`Unwrap`) | Callers use `errors.Is`, not string sniffing ‚Äî satisfies `design.md ¬ß11` error taxonomy. |
| `ResolveProviderEvent` wraps existing `internal/trigger.MatchTrigger` | Reuses the proven matcher; new package is purely additive over the existing trigger primitives. |
