# Required Outcomes
1. **WriteRefs** (`refs.go`) — implements `catalog-store.md` §3.D
   step-by-step:
   - D.1: `refs/sources/current.json` via `CompareAndSwap` with
     retry-on-`ErrConflict` (re-read, merge, re-CAS). Retry budget:
     16 attempts (matches the seq.lock retry budget for consistency).
     Exhaustion → `ErrRefStale` wrapping the last `statestore.ErrConflict`
     via the `fmt.Errorf("%w: %w", ...)` double-wrap pattern.
   - D.2: `refs/catalogs/current.json` — same.
   - D.3: If `refs.Source.Authoritative` (or equivalent flag in
     `SourceRef` — pick the field that means "this is the main ref"):
     `refs/sources/main.json` and `refs/catalogs/main.json`. If
     `RefUpdate` lacks an authoritative-flag field, propose a minimal
     extension at `ai/proposals/task-0032-spec-update.md` and FAIL the
     task.
   - D.4: If `refs.Source.Branch != ""` (and similarly for catalog):
     `refs/sources/branches/<sanitized-branch>.json` and
     `refs/catalogs/branches/<sanitized-branch>.json`.
   - D.5: If `refs.Source.PR != ""` (and similarly for catalog):
     `refs/sources/prs/<sanitized-pr>.json` and
     `refs/catalogs/prs/<sanitized-pr>.json`.
   - D.6: `refs/sources/latest.json`, `refs/catalogs/latest.json`
     (always written last).
   - When `RefUpdate.Source == nil`, skip every D.* that targets a
     source ref. When `RefUpdate.Catalog == nil`, skip catalog refs.
     Both nil → no-op success.
   - Initial write (no existing ref) goes through `CreateIfAbsent`,
     not `CompareAndSwap` (CAS with empty `oldRev` is undefined per
     statestore semantics — confirm from statestore source). On
     `ErrExists` with byte-identical body, success; otherwise fall
     through to the CAS retry loop.

2. **WriteGlobalIndexes** (`indexes.go`) — implements
   `catalog-store.md` §3.C:
   - C.1: `StateStore.Write(SourceGlobalIndexPath(srcKey), body)` —
     plain Write (overwrite OK). Skip if `updates.Source == nil`.
   - C.2: `StateStore.Write(CatalogGlobalIndexPath(srcKey, catKey),
     body)` — plain Write. Skip if `updates.Catalog == nil`.
   - C.3: For each component reachable from `updates.Catalog.Components`
     (or whichever field your `GlobalIndexUpdate` declares — confirm
     against `store.go` PR-1 surface):
     `CompareAndSwap(ComponentGlobalIndexPath(componentKey), oldRev,
     mergedBody)` with retry-on-`ErrConflict`. Loser must re-read
     and merge against the latest body, NOT clobber. Retry budget:
     16 attempts; exhaustion → `ErrRefStale`-equivalent (or a
     dedicated `ErrGlobalIndexStale` — your call, but be consistent
     with §6 taxonomy).
   - Component sort: deterministic, by component-key
     ascending — every test must assert order so a future map-range
     refactor breaks the build.

3. **AppendComponentEvent** (`events.go`) — implements
   `catalog-store.md` §3.D's history-event paragraph (lines 193-196):
   - Allocate `<seq>` via `CreateIfAbsent` on
     `<eventsDir>/seq.lock` containing the next sequence number.
   - On `ErrExists`: read current `seq.lock`, increment, retry. Up
     to 16 attempts. Exhaustion → typed `ErrEventSeqExhausted` (or
     reuse `ErrRefStale` if you want a single retry-exhausted
     sentinel — DOCUMENT the choice in the report and pick whichever
     keeps the taxonomy tightest).
   - Once allocated, write the event body at
     `ComponentHistoryEventPath(srcKey, catKey, name, seq, kind)`
     via `CreateIfAbsent` (events are immutable per spec). Body
     encoder: `catalogmodel.PrettyEncode` (consistency with PR-1
     body writes).
   - Fields validation: `ev.SourceSnapshotKey`,
     `ev.CatalogSnapshotKey`, `ev.ComponentName`, `ev.EventType`
     must be non-empty (or whatever the catalogmodel struct
     declares as required — read `events.go` in catalogmodel).
