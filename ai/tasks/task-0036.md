# Objective

Ship `cmd/orun/catalog.go` (root `orun catalog` command + dispatch) and
the first two subcommands:

1. **`orun catalog refresh`** — resolve the current workspace into a
   `(SourceSnapshot, CatalogSnapshot)` pair and persist it through
   `internal/catalogstore`, end-to-end:
   - `sourcectx.ResolveSourceSnapshot` → build `catalogresolve.
     ResolverInputs` → `catalogresolve.BuildCatalog` → assemble the
     persistence bundles → `catalogstore.WriteSourceSnapshot`,
     `WriteCatalogSnapshot`, `WriteRefs`, `WriteGlobalIndexes`.
   - Idempotent reuse: a byte-identical re-refresh prints the `↺ Catalog
     up to date` form and exits 0 (do not error on idempotent re-write).
   - Dirty-worktree banner is always printed when the workspace is dirty
     (`cli-surface.md` §2).
   - Flags: `--source <selector>` (alias `--catalog-source`),
     `--catalog-snapshot <key>`, `--strict` (alias `--catalog-strict`),
     `--no-infer`, `--json`, and `--sync` (Phase 2: prints `remote sync
     not configured` and exits 0 — no networking).
   - Exit codes per `cli-surface.md` §2 (0 created/reused, 1 validation
     error / any warning under `--strict`, 2 resolver internal error,
     3 statestore persistence failure).
2. **`orun catalog refs`** — enumerate every catalog ref (`main`,
   `current`, branches, PRs) with resolved source/catalog keys and the
   `authoritative` flag (`cli-surface.md` §8). `--json` supported.

Plus the shared CLI plumbing every later C5 command will reuse:
   - The `orun catalog` root with a subcommand index help page
     (`cli-surface.md` §12).
   - A `--catalog-source`/`--catalog-snapshot` → `catalogstore.
     RefSelector` parser (`current` | `main` | `branches/<name>` |
     `prs/<n>` | explicit `cat-<key>`/snapshot pin). One helper, reused
     by every subcommand.
   - The shared `--json` envelope writer producing the existing Orun
     envelope `{apiVersion: "orun.io/v1alpha1", kind, data, warnings}`
     (`cli-surface.md` §11) with `kind` ∈ `{CatalogRefreshResult,
     CatalogRefsResult}` for this PR.
   - Help-text fixtures for the three help pages shipped here
     (`orun catalog`, `refresh`, `refs`).

# Integration Notes

Two seams do not exist yet and are the crux of this PR — decide their
home deliberately and justify it in the report:

1. **Persistence-bundle assembly.** `BuildCatalog` returns a
   `CatalogView` (pure: `ResolvedCatalog` + `*CatalogSnapshot` +
   `[]*CatalogGraph`). The `catalogstore.Writer` needs `CatalogGraphs`,
   `CatalogLocalIndexes` (component / owner / system / domain / type
   bodies), `GlobalIndexUpdate` (`[]*ComponentGlobalIndex`), and
   `RefUpdate` (`SourceRef` + `CatalogRef` + branch/PR scope). There is
   no helper today that derives these bundles from a `CatalogView` +
   `WorkspaceState`. Build that as a tested pure helper (recommended:
   `internal/catalogresolve` for the index/graph bundles, since it owns
   the resolved data shapes; `internal/catalogstore` or the CLI for the
   `RefUpdate` scope which is source-key-driven). Do not bury this logic
   untested inside the cobra `RunE`.

2. **Ref enumeration.** The `Resolver` interface resolves *per selector*
   but cannot *list* refs. `orun catalog refs` needs to walk
   `refs/sources/*` + `refs/catalogs/*` (incl. `branches/` and `prs/`)
   and join them with `authoritative`. Add a typed enumeration method /
   helper (e.g. a `ListRefs(ctx)` on a small new reader, or a
   `catalogstore` function) over `statestore` directory reads — no raw
   `os`/`path/filepath` imports in `internal/catalogstore` (the existing
   no-raw-FS lint must stay green). Mirror the §4 path conventions; do
   not concatenate paths by hand.

Both seams are reused by later C5 PRs (`list` reads the same enumeration;
`refresh` is the only writer), so getting their shapes right here is the
leverage of this task. If you discover the spec under-specifies a bundle
field or a ref-listing shape, proceed with a documented narrow
assumption and file `ai/proposals/task-0036-spec-update.md` rather than
silently diverging.
