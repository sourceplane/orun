## Goal

Ship the hidden `orun state migrate` command and close M5.

The command walks legacy `.orun/plans/<hex>.json` and `.orun/executions/<id>/`
artifacts and surfaces them through the new revision-first layout per
compat-and-migration §5.1:

1. For each `.orun/plans/<checksum>.json`: synthesize a `system.migrated`
   trigger, derive its revision key, and (unless already on disk) write the
   revision-first triplet (trigger.json + revision.json + plan.json + index +
   refs) via `internal/revision.WriteRevision`.
2. For each `.orun/executions/<execID>/`: read the legacy `state.json` to
   recover its `planChecksum`, locate the matching revision (creating an
   `unknown-p<deadbeef>` revision on demand for orphans), then mirror the
   legacy `state.json` + `metadata.json` into
   `revisions/<revKey>/executions/<execKey>/` via
   `internal/executionstate.Bridge.MirrorRunnerOutput`. Persist a fresh
   `executionstate.CreateExecution` record so the execution shows up in the
   new layout's index/refs.
3. Print a summary (`revisions_created`, `executions_attached`, `orphans`).
4. Honor `--dry-run` (read-only — no writes, no exit-1 on orphans).
5. Exit non-zero only on per-item *errors* (continue processing; surface count
   on stderr at end). Orphans are NOT errors.

## Constraints

- Path policy: never construct state paths in callers. Route through
  `internal/statestore/paths.go` helpers (`LegacyExecutionsRoot`,
  `LegacyExecutionFilePath`, `LegacyExecutionDocPath`,
  `RevisionDir`, `RevisionDocPath`, etc.). The legacy `.orun/plans/`
  directory uses `internal/revision/legacy.go` helpers, which expose a
  package-internal API; the migrate command therefore lives in `cmd/orun`
  but delegates plan-hash discovery to a small new exported helper in
  `internal/revision` (e.g. `ScanLegacyPlanHashes`) so we don't grow
  `cmd/orun` with raw `plans/` literals.
- No new sentinel errors. Wrap the four existing `statestore` sentinels
  (`ErrInvalid`, `ErrExists`, `ErrNotFound`, `ErrConflict`) via `fmt.Errorf`.
- Strict JSON decode (`DisallowUnknownFields`) for any on-disk doc parsing.
- The `--persist-revision` flag was reserved by M5.b but never wired; this
  task does NOT wire it (M5.b/M5.c left it as a runtime no-op; migrate
  always persists when it acts). A note in the task report records this so
  M6 can decide whether to remove the reservation or wire it on `run`.
- Bridge mirrors only `state.json` + `metadata.json` per
  `bridgeMirroredFiles`. Re-using `Bridge.MirrorRunnerOutput` here keeps
  that contract intact.
- Idempotent: re-running `orun state migrate` produces zero new files. The
  revision-key derivation (`triggerKey + planHash8`) plus the
  CreateIfAbsent-on-index claim guarantees this.
- Coverage gates must hold: statestore ≥95.7%, revision ≥90.4%,
  executionstate ≥90.0%. New `cmd/orun` tests are not gated.
