## Objective
Stand up Phase 2's no-FS foundation by introducing `internal/catalogmodel`
(every persisted Phase 2 type per `specs/orun-component-catalog/data-model.md`,
plus canonical-JSON marshalers, sanitizers, and a committed JSON Schema for
`component.yaml`) and a `internal/sourcectx` types-only skeleton
(`model.go` / `keys.go` / `hash.go` shapes that C1 will populate). Extend
`make test-state-redesign` to gate the new packages with a coverage floor on
`catalogmodel` and a build/test gate on `sourcectx`. Phase 1 invariants and
coverage floors must hold byte-for-byte.

## PR Boundary
- New package `internal/catalogmodel`:
  - One file per data-model.md section (`source_snapshot.go`,
    `catalog_snapshot.go`, `component_manifest.go`, `component_yaml.go`,
    `graph.go`, `events.go`, `refs.go`, `indexes.go`, `entity_ref.go`,
    `sanitize.go`) holding the Go structs with **lowerCamelCase JSON tags
    matching `data-model.md` exactly**.
  - A canonical-JSON encoder (sorted keys, no whitespace, RFC 3339 / `Z`
    timestamps, deterministic map-key ordering) used for hashing inputs;
    a separate human-readable encoder (sorted keys, 2-space indent) used
    for persisted artifacts. Hashed inputs MUST go through the canonical
    encoder — no bare `encoding/json` for any hash input.
  - ID-prefix helpers (`src_`, `cat_`, `cmp_`) and key formatters per
    `identity-and-keys.md` §2–§4 (no resolution logic — pure construction +
    validation).
  - Sanitizer helpers (`SanitizeBranch`, `SanitizeComponentKey`,
    `SanitizeEventKind`, `ShortHex`) per `identity-and-keys.md` §12, total
    and panic-free.
  - `go generate ./internal/catalogmodel` emits a JSON Schema artifact under
    `internal/catalogmodel/schema/component-yaml.schema.json` (committed).
  - `testdata/` golden fixtures for every type with `decode → re-encode`
    roundtrip tests.
- New package `internal/sourcectx` skeleton:
  - `model.go` (struct definitions only, mirroring `data-model.md` §1
    `SourceSnapshot` shape but as the in-memory pre-persistence type),
    `keys.go` (`SourceSnapshotKey` type + parser/validator stub returning
    `ErrNotImplemented` for resolution paths), `hash.go` (constants /
    interface stubs for `Git`, `Clock`, `Filesystem` injection — no
    implementations).
  - At least one `_test.go` per file exercising the type signatures (build
    gate; resolver tests land in C1).
- `Makefile`: extend `test-state-redesign` to also run
  `go test -count=1 -race ./internal/catalogmodel/... ./internal/sourcectx/...`
  and assert a `>= 90 %` coverage gate on `internal/catalogmodel/...` (mirror
  the existing `internal/statestore` coverage gate shell). Per
  `test-plan.md` §1, sanitizers must be 100 % covered — assert via a
  package-level coverage check or a focused `_test.go` matrix that touches
  every sanitizer branch.
- `make verify-generated` target: regenerate the JSON Schema and `git diff`
  it; non-zero exit if the committed artifact drifts from generator output.
  Hook into CI after `make test-state-redesign` (no new workflow file —
  reuse the existing one).

## Constraints

   (Phase 2 data-model.md correctness invariant + C0 "done when").
2. All hashing inputs go through the canonical encoder; no bare
   `encoding/json` for any hash input. Document the encoder contract in a
   package-level Go doc comment.
3. Pin `pgregory.net/rapid v1.1.0` (already in `go.mod`) for property tests.
4. `lowerCamelCase` JSON tags must match `data-model.md` exactly. RFC 3339
   `Z` timestamps. ULID IDs use `oklog/ulid/v2` (already a Phase 1 dep).
5. Phase 1 invariants: do not rename or remove any Phase 1 field, do not
   lower any Phase 1 coverage floor. `make test-state-redesign` must keep
   passing the existing gates byte-for-byte.
6. JSON Schema is generator-output, not hand-written. `make verify-generated`
   must fail when the committed artifact drifts.
7. No secrets in fixtures or test output. No plaintext tokens.
