# Objective
Create `internal/catalogstore` and land the path-helper layer plus the body-write half of `Writer`:

1. `internal/catalogstore/paths.go` — every helper named in `specs/orun-component-catalog/catalog-store.md` §2, plus the matching `Validate*` siblings. Forward slashes only. No raw path concatenation; all helpers go through `path` (slash) joins. Helpers MUST NOT panic; invalid input returns `("", err)` from a sibling `Validate*` and the public helper either returns `(string, error)` or panics ONLY on a programmer error already screened by `Validate*`. Pick one shape and apply it consistently — recommended: each helper returns `(string, error)` and validates inline.

2. `internal/catalogstore/writer.go` — partial `Writer` implementation backed by `internal/statestore.StateStore`. This PR implements:

     - `WriteSourceSnapshot(ctx, src)` — step A: `CreateIfAbsent(SourceDocPath(srcKey), canonicalJSON(src))`. On `ErrExists` with byte-identical body → success. On `ErrExists` with different body → return `ErrSourceMismatch` wrapping the `statestore` error (preserve `errors.Is` for `statestore.ErrExists`).
     - `WriteCatalogSnapshot(ctx, src, cat, manifests, graphs, localIndexes)` — steps B.1 → B.4 in order:
         * B.1 each manifest via `CreateIfAbsent(ComponentManifestPath, body)`; mismatch → `ErrManifestMismatch`.
         * B.2 each graph kind in fixed order `dependencies, systems, apis, resources, owners` via `CreateIfAbsent(CatalogGraphPath, body)`.
         * B.3 the catalog doc via `CreateIfAbsent(CatalogDocPath, body)`; mismatch → `ErrCatalogMismatch`.
         * B.4 each catalog-local index via plain `Write` (overwrite is allowed; rebuildable).
       The writer MUST refuse to proceed if `cat.source.sourceSnapshotKey != src.sourceSnapshotKey` or the manifests' `source.sourceSnapshotKey` / `source.catalogSnapshotKey` don't match — return a typed pre-flight error (e.g. `ErrInputsInconsistent`). This is the writer-side mirror of the C3 invariant the resolver guaranteed in memory.

3. `internal/catalogstore/errors.go` — declare the error variables this PR uses: `ErrSourceMismatch`, `ErrCatalogMismatch`, `ErrManifestMismatch`, `ErrInputsInconsistent`. The remaining error variables (`ErrCatalogNotFound`, `ErrComponentNotFound`, `ErrRefStale`) belong to PR-3 / PR-2 and may be omitted here. All wraps preserve `errors.Is` against `statestore` sentinels (`ErrExists`, `ErrConflict`, `ErrInvalid`, `ErrNotFound`).

4. Public surface (`type Writer interface`, `type Resolver interface`, `type Store interface`, `func New(...)`) MUST be declared in this PR exactly as shown in `catalog-store.md` §1, but the `Resolver` methods and the not-yet-implemented `Writer` methods (`WriteRefs`, `WriteGlobalIndexes`, `AppendComponentEvent`) return a typed `ErrNotImplemented` (or panic with a clear "implemented in C4 PR-2/PR-3" message — pick one and document it). PR-2 and PR-3 will fill them in without changing the interface signatures.

5. `New(state statestore.StateStore) Store` — constructor that returns the in-progress implementation. No new dependencies beyond `internal/statestore` and `internal/catalogmodel`.

Canonical-JSON marshalling for every body MUST go through `internal/catalogmodel`'s canonical encoder (the same encoder used by C0/C2 hashing). Do NOT use `encoding/json` defaults for any body the writer hands to `statestore`.

# Integration Notes
- The writer's atomicity guarantees are inherited from `statestore`. Do not re-implement temp-file rename or CAS.
- Step B.4 (`Write` for local indexes) is intentionally non-CAS: local indexes are rebuildable. Do not "harden" them with CAS in this PR.
- `ErrInputsInconsistent` is the pre-flight version; the runtime guard against partial writes still relies on `CreateIfAbsent` returning `ErrExists` with mismatched bodies. Both signals are required: pre-flight catches programmer errors fast, runtime catches concurrent-writer divergence.
- The `Writer` interface is the public contract callers (C5 CLI, C6 plan, C7 run) will consume — finalize signatures here. PR-2/PR-3 fill in bodies but MUST NOT widen or rename methods.
